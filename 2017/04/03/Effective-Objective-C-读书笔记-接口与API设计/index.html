用前缀避免命名空间冲突

OC语言没有命名机制。所以在命名的时候很容易就重名的。会出现很多问题。这个时候需要为所有的名称都加上合适的前缀。可以与公司或者应用程序有关联的。但是苹果保留了“两字母前缀”的权利，所以自己命名前缀的时候，最好加三个字母的前缀。

注意：应用程序中的所有名称都应加上前缀。如果是分类，及一定要给分类和分类里面的方法加上前缀。还有就是类的实现文件中所有的纯C函数及全局变量也需要加上特定的前缀。如果是引入的第三方库，最好把第三方库的前缀换成自己的前缀

尽量使用不可变对象

设计类的时候尽量使用属性封装数据，使用属性的时候，可将其声明为“只读”。如果不声明就默认是“既可读又可写”，就是可变的。一般情况下我们要建模的数据未必需要改变，如果源自一项只读的网络服务，及时修改了也不能传给服务器。所以还是用不可变的。

这下如果有人试着改变属性值，那么编译的时候就会报错。属性可以读，但是不能写入。可能有这么个疑问，这些属性都没有设置方法，还要内存管理语义干嘛？如果指定了内存管理语义，以后把它改为可写可读属性时就会简单点。

使用清晰而协调的命名方式

尽量写明一个方法或是变量名的用途，英文不怕长，需要清晰表达出意思。还有使用“驼峰式大小写命名法”。

NSString这个类就展示了一套良好的命名习惯：

  • +string
    • 工厂方法，用于创建新的空字符串，清楚的描述了返回值类型；
  • +stringWithString
    • 工厂方法，根据某字符串创建出与之内容相同的新字符串，清楚的描述了返回值类型；
  • +localizedStringWithFormat:
    • 工厂方法，根据特定格式创建出新的“本地化字符串”，返回值还是“字符串”，只不过是一种经过本地化处理的特殊字符串；
  • -lowercaseString
    • 字符串中大写字母都转为小写，该方法不会直接修改调用该方法的字符串，而是另外创建一个字符串。返回值也是字符串类型；
  • -intValue
    • 将字符串解析为整数。
  • -length
    • 获取字符串长度
  • -lengthOfBytesUsingEncoding:
    • 若字符串是以给定的编码格式（ASCII、UTF8、UTF16等）来编码的，则返回其字节数组的长度。
  • -getCharacters: range:
    • 获取字符串中给定范围内的字符
  • -hasPrefix:
    • 判断本字符串是否以另一个字符串开头
  • -isEqualToString:
    • 判断两个字符串是否相等

给方法命名时注意事项：

  • 如果方法名的返回值是新创建的，那么方法名的首个词应该是返回值的类型，除非前面还有修饰语，如lowercaseString。属性的存储方法命名就不是这样的，因为一般认为这些方法不会创建新对象。
  • 把表示参数类型的名词放在参数前面
  • 若方法要在当前对象上执行操作，那么就应该包含动词；若执行操作时还需要参数，则应该在动词后面加上一个或多个名词
  • 不要使用str这种简称，应该用string全称
  • Boolean属性前面加上is或者has
  • 将get这个前缀留给那些借由“输出参数”来保存返回值的方法。比如说，把返回值填充到“C语言式数组”里的那种方法就可以使用这个词做前缀。

类与协议的命名

应该为类和协议的名称加上前缀，避免命名空间冲突。说到底一点就是命名方式应该协调一致。

为私有方法加前缀

一方面可以在调试的时候很容易判断出是内部方法还是外部方法出错；另一方面便于修改方法名和方法签名。因为公共方法来说，修改方法名，在调用的地方也要同步修改才行。一般可以在方法前加上p_。

注意不要单用一个下划线做私有方法的前缀，因为这种所发是预留给苹果公司的。

理解Objective-C错误类型

异常：

  • 自动引用计数(ARC)在默认情况下不是“异常安全的”。具体来说，这意味着：如果抛出异常，那么本应在作用域末尾释放的对象现在却不会自动释放了。如果想生成“异常安全”的代码，可以通过设置编译器的标志来实现。：-fobjc-arc-exceptions。

OC语言对异常的操作：只在及其罕见的情况下抛出异常，异常抛出后无需考虑恢复问题，而且应用程序此时也应该退出。这就是说，不用再编写复杂的“异常安全”代码了。

异常只应该用于及其严重的错误，比如说，编写了某个抽象基类，它的正确用法是先从中继承一个子类，然后使用这个子类。如果有人直接使用这个抽象基类，那么可以考虑抛出异常。但是OC没有将某个类表示为抽象类的功能，所以可以再那些子类必须覆写的超类方法中抛出异常。

__
异常时处理严重错误的时候使用的，而不严重的错误可以使用NSError或者返回nil/0。

NSError的用法更加灵活，因为经由此对象，可以把导致错误的原因回报给调用者。NSError对象里面封装了三条信息：

  • Error domain（错误范围，字符串类型）
    • 这个是指明错误的根源。
  • Error code （错误码，整数类型）
    • 发生的何种错误
  • User info （用户信息，字典类型）
    • 有关此错误的额外信息
